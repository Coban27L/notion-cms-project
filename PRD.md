# 노션 CMS 견적서 PRD

## 프로젝트 개요

노션(Notion)을 CMS로 활용하여 견적서를 작성하고, 클라이언트가 고유 링크를 통해 웹에서 확인 및 PDF로 다운로드할 수 있는 시스템입니다. 별도의 관리자 UI 없이 노션에서 모든 콘텐츠를 관리하며, Next.js가 노션 API를 통해 데이터를 가져와 렌더링합니다.

---

## 사용자 여정 플로우

**견적서 발행 흐름 (관리자)**
노션 DB 작성 → 고유 링크 생성 → 클라이언트에게 링크 전달

**견적서 확인 흐름 (클라이언트)**
고유 링크 접속 → 견적서 상세 페이지 → PDF 다운로드

**관리자 확인 흐름 (관리자)**
관리자 로그인 → 대시보드 → 견적서 목록 → 견적서 상세 페이지 → PDF 다운로드

## 기능 명세

### 1. 핵심 기능

| ID | 기능명 | 설명 | 사용자 기본 니즈 | 관련 페이지 |
|----|--------|------|----------------|------------|
| **F001** | 노션 견적서 조회 | 노션 API를 통해 견적서 데이터를 가져와 렌더링 | 노션에서 입력한 내용을 웹에서 보고 싶다 | 견적서 상세 페이지 |
| **F002** | 견적서 목록 조회 | 전체 견적서 목록을 카드 형태로 표시 | 작성한 견적서 전체를 한눈에 관리하고 싶다 | 대시보드 |
| **F003** | 고유 링크 견적서 접근 | 토큰 기반 고유 URL로 클라이언트가 인증 없이 견적서 접근 | 클라이언트에게 견적서 링크를 공유하고 싶다 | 견적서 상세 페이지 |
| **F004** | PDF 다운로드 | 견적서를 PDF 파일로 변환하여 다운로드 | 견적서를 PDF로 저장하거나 인쇄하고 싶다 | 견적서 상세 페이지 |
| **F005** | 관리자 인증 | 이메일/비밀번호로 관리자 로그인, 대시보드 접근 보호 | 외부인이 전체 견적서 목록에 접근하지 못하게 하고 싶다 | 로그인 페이지, 대시보드 |

### 2. MVP 필수 지원 기능

- **노션 DB 스키마 정의**: 견적서 번호, 클라이언트명, 항목, 금액, 상태, 공유 토큰 필드를 포함하는 노션 DB 구조
- **공유 토큰 생성 로직**: 각 견적서마다 UUID 기반 토큰을 노션 DB에 저장하고, 해당 토큰으로 URL 생성
- **반응형 디자인**: 모바일/태블릿/데스크톱 모두에서 견적서가 올바르게 렌더링

### 3. MVP 이후 기능 (제외)

- 견적서 열람 여부/횟수 추적
- 클라이언트 이메일 발송 기능
- 견적서 승인/거절 상태 변경 (클라이언트 액션)
- 다국어 지원
- 커스텀 브랜딩 설정 (로고, 색상)
- 견적서 템플릿 관리
- 알림 기능 (Slack, 이메일)

## 데이터 모델

노션 DB가 primary 데이터 소스이며, 별도의 자체 DB는 MVP에서 사용하지 않습니다. 관리자 인증을 위해 환경 변수 기반 단순 인증을 사용합니다.

### 노션 DB 스키마 (견적서)

| 필드명 | 설명 | 타입/관계 |
|--------|------|----------|
| title | 견적서 번호 (예: QT-2024-001) | Title |
| client_name | 클라이언트 이름 또는 회사명 | Text |
| status | 견적서 상태 | Select (발행/승인/취소) |
| total_amount | 견적 총액 | Number |
| issued_date | 발행일 | Date |
| valid_until | 견적 유효기간 | Date |
| share_token | 고유 공유 토큰 (UUID) | Text |
| items | 견적 항목 상세 내용 | Rich Text (JSON 형식으로 입력) |
| notes | 비고 / 특이사항 | Rich Text |

### 견적 항목 구조 (items 필드 내 JSON)

```
[
  { "name": "품목명", "quantity": 수량, "unit_price": 단가, "amount": 금액 }
]
```

## 환경 변수 구성

MVP에서 필요한 환경 변수 목록입니다.

```
NOTION_API_KEY=         # 노션 통합 API 키
NOTION_DATABASE_ID=     # 견적서 노션 DB ID
ADMIN_EMAIL=            # 관리자 이메일 (단일 계정)
ADMIN_PASSWORD=         # 관리자 비밀번호 (해시 저장 권장)
NEXTAUTH_SECRET=        # next-auth 세션 서명 키
NEXTAUTH_URL=           # 배포 URL
```
