# 노션 CMS 견적서 시스템 개발 로드맵

노션을 CMS로 활용하여 견적서를 발행하고, 클라이언트가 고유 링크로 웹에서 확인 및 PDF로 다운로드할 수 있는 시스템입니다.

## 개요

노션 CMS 견적서 시스템은 **1인 프리랜서 및 소규모 팀**을 위한 **노션 기반 견적서 발행 및 공유 플랫폼**으로 다음 기능을 제공합니다:

- **노션 기반 콘텐츠 관리**: 노션 DB에서 견적서를 작성하면 웹에서 자동 렌더링
- **고유 링크 공유**: UUID 토큰 기반 URL로 클라이언트가 인증 없이 견적서 열람
- **PDF 다운로드**: 견적서를 PDF로 변환하여 저장 및 인쇄 가능
- **관리자 대시보드**: 관리자 전용 레이아웃 기반 견적서 관리 허브

## 개발 워크플로우

1. **작업 계획**
   - 기존 코드베이스를 학습하고 현재 상태를 파악
   - 새로운 작업을 포함하도록 `ROADMAP.md` 업데이트
   - 우선순위 작업은 마지막 완료된 작업 다음에 삽입

2. **작업 생성**
   - `/tasks` 디렉토리에 새 작업 파일 생성 (예: `015-admin-layout.md`)
   - 고수준 명세서, 관련 파일, 수락 기준, 구현 단계 포함
   - API/비즈니스 로직 작업 시 "## 테스트 체크리스트" 섹션 필수 포함

3. **작업 구현**
   - 작업 파일의 명세서를 따름
   - Playwright MCP를 활용한 E2E 테스트 수행
   - 각 단계 후 작업 파일 내 단계 진행 상황 업데이트

4. **로드맵 업데이트**
   - 완료된 작업을 ✅로 표시하고 `/tasks/XXX-xxx.md` 참조 추가

### Phase 7: 품질 개선 및 버그 수정

> **목표**: PDF 다운로드 시 한글 글자가 렌더링되지 않는 핵심 버그를 해결하고, 접근성과 회귀 테스트 커버리지를 강화합니다.

- ✅ **Task 021: PDF 한글 폰트 렌더링 버그 수정**
  - 폰트 파일을 base64 데이터 URL로 변환하여 registerFontsAsync() 함수에 전달
  - Noto Sans KR Regular/Bold 두 weight 모두 등록
  - 비동기 폰트 로딩 보장 (fontRegistered 플래그로 캐싱)
  - RFC 5987 HTTP 헤더 인코딩으로 파일명 처리
  - **테스트 통과**: 한글 클라이언트명/품목명이 포함된 견적서 PDF 정상 렌더링 ✅
  - **수락 기준 충족**: 모든 한글 텍스트가 PDF에서 시각적으로 정상 렌더링 ✅

- ✅ **Task 022: PDF 레이아웃 개선**
  - 발행일/유효기간 한국 날짜 포맷 구현 (`yyyy년 MM월 dd일`) ✅
  - 통화 포맷 구현 (`₩` + 천 단위 콤마, Intl.NumberFormat) ✅
  - 다중 페이지 지원 (항목 10개 단위로 페이지 분할) ✅
  - 합계 영역을 마지막 페이지에만 표시 ✅
  - 비고(notes) 섹션을 마지막 페이지에만 표시 ✅
  - 다중 페이지 문서에 페이지 번호 표시 (`페이지 1 / 2`) ✅
  - 테이블 pageBreakInside: 'avoid' 적용 ✅
  - **테스트 통과** (13/13): 
    - 단일 페이지 PDF (2개 항목) ✅
    - 다중 페이지 PDF (15개 항목 → 2페이지 생성) ✅
    - 한국 날짜/통화 포맷 정확도 ✅
    - 모바일/태블릿 반응형 PDF 다운로드 ✅
  - **수락 기준 충족**: 모든 레이아웃 개선 사항 구현 완료 ✅

- ✅ **Task 023: 접근성(a11y) 및 키보드 네비게이션 개선**
  - Skip to main content 링크 구현 ✅
  - 모든 아이콘 버튼에 ARIA 라벨 추가 ✅
  - 포커스 스타일 개선 (focus:ring-2, focus:ring-blue-500) ✅
